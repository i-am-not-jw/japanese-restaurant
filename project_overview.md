# 🍱 로컬 맛집 내비게이터 (Japan Local Restaurant Scraper & Notion Publisher)

본 프로젝트는 일본의 양대 플랫폼인 **'타베로그(Tabelog)'**와 **'구글 맵(Google Maps)'**의 데이터를 실시간으로 교차 검증하여, 광고성 리뷰가 아닌 **'진짜 현지인 입맛에 맞는 숨겨진 맛집'**만을 큐레이션하고 Notion 데이터베이스로 자동 출판하는 데이터 파이프라인입니다.

---

## 🏗️ 1. 아키텍처 및 철학 (Architecture & Philosophy)

이 프로젝트는 안정성과 확장성을 극대화하기 위해 **3-Layer Architecture (지시 - 조정 - 실행)** 구조를 채택했습니다. AI의 환각(Hallucination) 현상을 차단하면서도, 필요할 때만 AI의 언어 능력을 활용하여 결정론적 세계(Deterministic World)와 생성형 세계(Generative World)를 안정적으로 결합했습니다.

- **Layer 1: Directive (기획 및 목적)**: `directives/routine_collection_sop.md` 등의 SOP 파일로 파이프라인의 기준, 태그 맵핑, 필터링 수치 등의 룰셋을 정의합니다.
- **Layer 2: Orchestration (결정 및 라우팅)**: `execution/daily_orchestrator.py`가 일본의 인구수 기준 Top 대도시부터 주요 소도시(Tier 2/3)까지 순회하며, 여러 스크립트를 연결하고 API 제한(Rate Limit 429) 발생 시 우아하게 종료/지연시키는 통합 관리자 역할을 수행합니다.
- **Layer 3: Execution (스크립트 처리)**: 타베로그 크롤러, 구글 맵 크롤러, Notion 퍼블리셔 등 개별 기능을 전담하는 모듈화된 파이썬 스크립트입니다.

## ⚙️ 2. 핵심 알고리즘 및 데이터 수집 프로세스

### Phase A. 타베로그 데이터 추출 (`tabelog_lookup.py`)
타베로그는 '일본 미식가들의 냉정한 평가 공간'으로, 높은 신뢰도를 지니고 있습니다.
1. **트렌드 순 정렬 기반 탐색:** 특정 지역의 검색 결과 중 `SrtT=trend` (트렌드 순) 옵션을 활용하여 최근 방문자가 급증한 '지금 뜨는 집'을 1차로 확보합니다.
2. **세부 정보 Parsing:** 식당 상세 메뉴, 연락처, 주소 영역에서 다음의 구체적 정보를 수집합니다.
    - **영업시간:** 요일, 정기휴일, 라스트 오더 등을 수집.
    - **결제수단:** "카드만", "현금만(Cash Only)", "간편결제 가능" 등 관광객 필수 정보를 태그화.
    - **교통수단 (역, 버스):** 
        - 정규식을 활용해 역 이름과 '도보 N분', '거리 M미터'를 파싱.
        - **위키백과 API(ja.wikipedia.org)**를 호출하여 일본어 철도역 한자를 오류나 환각 없이 "100% 공식 한국어 철도 명칭"으로 번역.
        - 버스 정류장 등 복잡한 비정형 구문에 한해서만 Gemini AI를 호출하여 자연스러운 한국어로 문맥 번역 및 소요 시간 삭제 등 데이터 정제.
3. **최신 리뷰 검증 (Recency):** 2년 전 별점 5점짜리 가게가 아닌 **우리가 지금 갈 맛집**을 찾기 위해, 최근 수동 리뷰의 '방문 연/월'을 반드시 수집합니다. 

### Phase B. 구글 맵 교차 검증 (`google_maps_lookup.py`)
현지 미식가의 평점(타베로그)에 **'대중들의 실시간 퀄리티 체크(구글 맵)'**를 교집합 시킵니다.
1. **Playwright 활용:** Headless 브라우저를 띄워 구글 맵에 직접 접근합니다.
2. **자바스크립트 DOM 파싱 리버스 엔지니어링:** 구글의 안티 봇 시스템을 우회하여 평점과 '리뷰 개수'를 동적 Javascript evaluation으로 강력하게 긁어옵니다.
3. **'최신순' 강제 정렬 동적 제어:** 기본 정렬이 아닌 **"최신순(Newest)"** 탭을 클릭해 가장 최근 5개의 생생한 리뷰 텍스트와 작성 시기(몇 시간 전, 며칠 전, 몇 개월 전)를 확보합니다. (예전 평점 대비 '지금 퀄리티'가 유지되는지 핵심 증거물 역할).

### Phase C. 데이터 정제 및 AI 요약
1. **필터링 조건 충족 여부:** 
   - 타베로그 평점 기준치(보통 3.5 이상).
   - 구글 맵 리뷰 50개 이상, 평점(4.0 이상) 등 엄격한 교집합 만족 식당만 다음 단계로 이동.
2. **Gemini AI 리뷰 요약:** 
   - 양쪽 플랫폼에서 긁어온 현지인들의 리뷰 텍스트 수백여 자를 Gemini API에 통과시킵니다.
   - 현지인들의 분위기, 극찬하는 메뉴, 줄 서는 정도, 접객 태도 등 단순 번역기론 알 수 없는 **'문맥적 뉘앙스(진짜 로컬맛집 여부)'**를 요약합니다.
3. **등급/뱃지 심사 시스템:** 
   - 평점과 리뷰 수를 상대적으로 연산해 `🦄 0.1% 레전드 맛집`, `🏆 현지인 인증맛집`, `💡 숨겨진 맛집` 등의 자동 뱃지 태그를 부여합니다.

## 📝 3. 데이터 검수 및 출판 (Data Staging & Upsert)

### Step 1. Staging (사전 검수 도입)
데이터 오염을 막기 위해 추출된 1차 무결점 데이터를 곧장 출판하지 않고, `/tmp/antigravity_tmp/staged_restaurants.csv`와 같은 **임시 CSV** 형태로 저장시킵니다. 사용자는 데이터를 훑어보고(Review), 승인하는 역할만을 수행합니다 (`export_to_csv.py`).

### Step 2. Upsert 기반 스마트 업로드 (`publish_from_csv.py`, `notion_publisher.py`)
사용자의 업로드(Publish) 허가가 떨어지면 Notion API를 호출합니다.
1. **중복 검사 (Deduplication):** `tabelog URL`을 고유 키(Primary Key)로 설정. DB 쿼리를 통해 이미 등록된 식당인지 파악.
2. **Upsert(업데이트 및 삽입) 처리:**
    - 기존 식당이라면: 페이지 ID를 획득하여 `PATCH /v1/pages/{id}` 로 평점, 리뷰 수, 최신 요약본 등만 최신화 (리뷰 트래킹 효과).
    - 새로운 식당이라면: 전체 속성과 이미지 썸네일을 담아 `POST /v1/pages` 
3. **도도부현(지자체) 공식 깃발 자동 주입:** 
   - `prefecture_flags.json`의 47개 일본 지자체 CDN 이미지를 조회하여 해당 지역의 공식 깃발을 노션 페이지의 썸네일/아이콘(Icon)으로 자동 삽입하여 시각적 완성도 증대.

## 📊 4. 구축된 Notion 데이터베이스 스키마

노션 데이터베이스는 마케터 및 엔드유저(구독자 파이프라인 등)가 직관적으로 볼 수 있게 설계되었습니다.

| 컬럼 / 속성명 | 데이터 모델 타입 | 설명 및 역할 |
|---|---|---|
| **제목** | Title | 맛집 이름 명칭 (원어 및 한국어 혼용) |
| **썸네일 (아이콘)** | URL Image | `자동 아이콘 주입 기술`로 해당 지역 지자체의 공식 깃발 표기 |
| **장소** | Rich text | 구글 맵 기준 도로명 + 상세 지역 (복사하기 편하게 제공) |
| **교통** | Rich text | "🚇 다카다노바바역에서 도보 5분", "🚌 호쿠테쓰 가나자와 버스" 등 (결정론적 / AI 혼합 매핑 기술) |
| **영업 시간** | Rich text | 라스트 오더, 정기휴일 등이 자연스러운 한국어로 서술됨 |
| **태그** | Multi-select | `[요리]`, `[뱃지 평점]`, `[지역]`, `[결제수단]` 우선순위대로 정렬 |
| **tabelog 평점** | Number | (마의 3.5 기준 통과 여부) |
| **tabelog 최신 리뷰** | Date | (이 식당이 최근 1개월 내 살아있는지 생존 트렌드 증명) |
| **구글맵/타베로그 리뷰 수** | Number | (신뢰도 검증용 군중 참여 지표) |
| **평점 비교** | Formula | `(타베로그 점수 - 구글맵 점수)`의 편차로 '대중픽 vs 미식가픽' 성향 확인 역산 |
| **요약** | Rich text | Gemini AI가 한글로 정리한 '신뢰할 만한 추천 사유' 1~2줄 |
| **관련 URL** | URL | 타베로그 및 구글 맵 다이렉트 랜딩 링크 |

## 🎯 5. 스레드 / 인스타그램 홍보 픽(Pitch) 전략: "우리가 다르게 하는 것"

위와 같은 정교한 백엔드 파이프라인 덕분에, 우리는 다음과 같은 메시지를 던질 수 있습니다:

1. **"리뷰가 최근이 아니면 거릅니다."** 단순히 5년 전 유명했던 가게의 별점 4.5에 속지 마세요. 우리 리스트는 '최신순 정렬' 데이터베이스만 읽어 어제자 퀄리티를 대조합니다.
2. **"오직 외국인이 아닌 '현지 일본인' 중심입니다."** 미식의 제국 타베로그의 살인적인 평점 구간(Top 3%)을 베이스로 합니다.
3. **"데이터의 이중 잠금."** 전문가적 시선의 '타베로그'와 최신 대중 반응인 '구글 맵'의 교집합. 두 깐깐한 세계에서 모두 살아남은 0.1% 레전드만 노출합니다.
